# 数据源调研结论

> 日期：2026-07-02　｜　方法：多源深度调研 + 编号 020608 实测验证

## 结论先行：两条腿走路，都免费

| 用途 | 数据源 | 获取方式 |
|---|---|---|
| **盘中实时估值**（当日涨幅、估算净值） | 天天基金 JSONP 接口 `fundgz.1234567.com.cn` | `GET https://fundgz.1234567.com.cn/js/<基金代码>.js` |
| **收盘真实净值**（算持仓真实盈亏） | AKShare（底层同为东财，免费 MIT） | `fund_open_fund_daily_em` 等，收盘后 16:00–23:00 更新当日净值 |

## fundgz 接口实测（2026-07-02）

请求：`https://fundgz.1234567.com.cn/js/020608.js`

返回：
```
jsonpgz({"fundcode":"020608","name":"南方中证机器人ETF发起联接C",
"jzrq":"2026-07-01","dwjz":"1.5946","gsz":"1.5622","gszzl":"-2.03",
"gztime":"2026-07-02 15:00"});
```

字段含义：

| 字段 | 含义 | 产品用途 |
|---|---|---|
| `fundcode` | 基金代码 | 主键 |
| `name` | 基金名称 | 展示 |
| `jzrq` | 上一交易日净值日期 | — |
| `dwjz` | 昨日单位净值 | 反推份额、盈亏基线 |
| `gsz` | 今日盘中估算净值（约每分钟更新） | 展示 |
| `gszzl` | 今日估算涨跌幅 % | **这就是"当日涨幅"，字段直读** |
| `gztime` | 估值时间 | 展示 |

## 必须记住的坑

1. **估值 ≠ 真实净值**。`gsz` 是第三方算法估算，含 **QDII / 港股 / 债券**的基金偏差大、
   部分基金无估值。产品逻辑：盘中看 `gszzl`（参考），**收盘后用官方 `dwjz` 重算真实盈亏**。
   对 QDII/债基类在 UI 标注"估值仅供参考"。
2. **非公开接口会封 IP**。AKShare 高并发抓东财有封 IP 记录（约 5 小时）。
   对策：**定时批量拉取 + 本地缓存**，用户读缓存，不要每次刷新都实时打接口。
3. **AKShare 与 fundgz 底层同为东方财富**，并非真正异构冗余；东财整体大改会同时失效。
   真正异构备用（新浪财经基金接口）本轮未取得强证据，自用场景可暂不投入。

## 被排除的方案（及原因）

| 方案 | 排除原因 |
|---|---|
| Tushare | 基金接口需 2000 积分（约 200 元/年），小用量过度投入 |
| JQData / Wind / 聚源 | 面向机构，需付费 / 鉴权 |
| 雪球（pysnowball） | 需手动维护 token/cookie，依赖未文档化端点，不稳 |
| 基金业协会 AMAC | 只有行业汇总统计，无单只基金净值/估值 |
| dayfund.cn | 无 API，只能爬 HTML，解析成本高 |

## 合规判断（非正式法律意见）

- 爬取非公开接口属灰色地带，AKShare 官方声明"数据仅供学术研究"。
- **几个朋友私享 + 低频 + 不商业化对外 → 风险很低。**
- 红线：一旦公开注册 / 收费 → 变为"对外提供服务"，风险显著上升。
- 原则：控制频率、不绕过反爬技术障碍、不对对方产品造成不当干扰。
