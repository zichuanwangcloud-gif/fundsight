# PRD-08 指数估值（PE / PB 历史分位 / 股息率 / 指数温度）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P3　｜　状态：评估型立项（数据源合规待评估）
>
> 主题：指数/ETF 定投的核心决策依据——看估值不看净值　｜　分类：抓取层扩面（合规敏感）

## 一、背景与价值

对指数和 ETF 联接（平台种子基金里就有机器人 ETF、银行 ETF、半导体 ETF、白酒指数、
恒生科技、纳指 100 多只），**"看估值不看净值"是定投常识**——定投看 PE 历史分位
（贵还是便宜），不看涨了多少。

**问题**：`fund_profile` 只有收益率（`syl_*`），没有估值字段。用户定投指数基金
缺少最核心的决策依据。

**价值**：补指数估值历史分位，让定投有"贵贱"锚。

> **⚠️ 本 PRD 为评估型立项**：指数估值数据源（PE/PB 序列）的合规边界比净值紧、
> 结构复杂，**必须先完成数据源评估子任务并复评合规红线，方可正式进入实施**。
> 当前标记 P3，不早于 PRD-01~07。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 指数定投者 | 看当前 PE 历史分位，分位低定投加额、分位高减额或止盈 |
| ETF 持有者 | 看股息率辅助判断高估/低估 |
| 选基者 | 按估值温度筛选处于低位的指数基金 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 指数 PE/PB 当前值 + 历史分位 | 当前 PE 在近 N 年的分位 % | P0 |
| 股息率 | 指数股息率 | P1 |
| 指数温度 | PE 分位简化版（0-100） | P1 |
| 估值分位走势（点状） | 近 1 年每月分位点（不画连续曲线） | P2 |

## 四、详细需求

### 4.1 估值表

```sql
CREATE TABLE IF NOT EXISTS index_valuation (
    index_code  TEXT NOT NULL,     -- 指数代码（如 000300 / HSTECH）
    trade_date  TEXT NOT NULL,
    pe_ttm      REAL,              -- 滚动市盈率
    pb          REAL,
    dividend_yield REAL,           -- 股息率 %
    PRIMARY KEY (index_code, trade_date)
);
CREATE TABLE IF NOT EXISTS fund_index_map (
    fund_code   TEXT PRIMARY KEY,
    index_code  TEXT                -- 该基金跟踪的指数
);
```

### 4.2 历史分位

- 取该指数近 5 年（可配置 `window=5y`）每日 `pe_ttm`，算当前 pe 在其中的百分位。
- `percentile = 当前 pe 在历史序列中的排名 / 总数 × 100`。
- 标签：< 20% 低估、20-50% 合理偏低、50-80% 合理偏高、> 80% 高估。
- PB 分位同理（可选返回）。

### 4.3 指数温度

- 温度 = PE 分位与 PB 分位的加权（默认各 50%），简化为单指标 0-100。
- 仅 PE 有时退化为 PE 分位。

### 4.4 基金 → 指数映射

- `fund_index_map`：指数/ETF 联接基金映射其跟踪指数；主动基金无映射，返回 null。
- 映射数据来源：东财基金分类自带跟踪指数字段。

## 五、数据来源与依赖

- **指数估值序列**：东财指数估值接口 / AKShare `index_value_hist_funddb` 等，
  **数据源稳定性与合规边界待评估**，收敛到 `backend/datasource/index_valuation.py`。
- 低频：估值日更即可（日频，但与净值同源同频，不新增频次风险）。
- **合规**：指数估值数据的版权与使用边界比净值紧，**公开/收费前须重走合规评估**。
  当前自用私享范围内评估可行性。

## 六、API 设计

```
GET /api/fund/{code}/valuation?window=5y
```
响应：
```json
{
  "fund_code": "020608",
  "index_code": "930009",
  "index_name": "中证机器人指数",
  "pe_ttm": 45.2,
  "pe_percentile": 62.3,
  "pb": 5.8,
  "pb_percentile": 58.1,
  "dividend_yield": 0.3,
  "temperature": 60.2,
  "label": "合理偏高",
  "note": null
}
```
主动基金（无映射）：`{"index_code": null, "note": "该基金非指数/ETF，无估值指标"}`。

## 七、前端交互

- 指数/ETF 详情页新增"估值"卡片：PE 当前 + 分位进度条 + 温度 + 标签。
- 主动基金详情页不显示估值卡（无意义）。
- 市场页指数类列表加"PE 分位"列，可排序筛低估。
- 不画估值连续走势曲线（红线）；分位用进度条 + 当前点呈现。

## 八、验收标准

1. 沪深300 ETF 的 PE 分位与理杏仁/蛋卷估值数据误差 ≤ 5 个百分位。
2. 主动基金返回 null + 提示，不报错。
3. 历史样本 < 1 年时标注"样本不足，分位仅供参考"。
4. 标签阈值正确：< 20 低估、> 80 高估。
5. `bash scripts/test.sh` 全绿。

## 九、范围边界（刻意不做，红线敏感）

| 项目 | 状态 |
|---|---|
| 估值连续走势曲线 | 不画（走势图红线，只给分位进度条） |
| 自定义指数组合估值 | 不做（写死基金→指数映射） |
| 估值预测 / 目标 PE | 不做（越界到预测） |
| **公开/收费提供估值数据** | **绝不做（合规红线，须重走评估）** |
