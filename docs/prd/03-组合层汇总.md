# PRD-03 组合层汇总（总资产 / 配置占比 / 集中度 / 相关性 / 组合回撤）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P0　｜　状态：待立项
>
> 主题：让"看板"名副其实——从管理自选到管理组合　｜　分类：组合看板做深

## 一、背景与价值

当前所有 API 都是**单基金维度**：`holding` 逐条、`transactions` 逐基金、
`holdings_ext` 只返一个 code 列表。用户打开"看板"，最想一眼看到的信息——
**总资产多少、整体盈亏多少、钱都压在哪儿**——平台给不出来。

**问题**：一个"看板"看不出组合全貌。用户持仓白酒 + 食品饮料 + 消费，看似分散
三只，实则一个赛道；60% 仓位压在一只上也没人提醒。

**价值**：基于已落库的 holding + transaction + nav_history，纯内部聚合出组合层视图。
让平台从"管理我的自选"升级为"管理我的组合"。零新增外部数据源。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 自用主理人 | 打开首页总览条，一眼看总资产、总盈亏、配置饼图、最大单基金占比 |
| 配置复盘 | 发现"以为自己分散了，实则重仓白酒一个赛道" |
| 风控 | 看组合层最大回撤——我的组合历史上最惨跌过多少 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 组合总览（总资产/总盈亏/加权收益率） | 按持仓份额聚合各基金盈亏 | P0 |
| 资产配置分布 | 按 fund_type 8 大类占比 | P0 |
| 持仓集中度 | 单基金占比 TOP、防过度集中 | P0 |
| 相关性矩阵 | 持仓基金间相关系数，识破"伪分散" | P1 |
| 组合层最大回撤 | 合成组合日净值序列算回撤 | P1 |

## 四、详细需求

### 4.1 组合总览

- 遍历当前用户 `holding` 全部持仓，对每只调 `compute_position` 得份额 + 成本。
- 每只市值 = `shares × latest_nav`（latest_nav 取 `fund_nav_history` 最新一条，
  无则用 `fund_quote.nav`/`dwjz` 兜底）。
- 汇总：`total_market_value`、`total_cost`、`total_pnl = 总市值 − 总成本`、
  `total_return_pct = total_pnl / total_cost × 100`。
- 加权收益率：`Σ(单基金盈亏) / Σ(成本)`，即钱加权（不是简单算术平均）。

### 4.2 资产配置分布

- 每只持仓查 `fund_list.fund_type`，用 `market.py` 的 `_category_case_sql`
  归入 8 大类（复用现有归类优先级，保证口径一致）。
- 按**市值**（非份额、非只数）算占比 → 配置饼图。
- 同时输出金额，便于"股票类 X 万 / 债券类 Y 万"展示。

### 4.3 持仓集中度

- 单基金市值占比降序，标 TOP1 占比是否 > 40%（预警阈值，灰字提示"集中度偏高"）。
- 可选：前 3 大持仓合计占比（CR3）。

### 4.4 相关性矩阵（P1）

- 取各持仓基金近 1 年 `equity_return_adj`（复权后，依赖 PRD-02）日序列。
- 两两 Pearson 相关系数。仅对样本重叠 ≥ 60 个交易日的基金对计算，否则 null。
- 前端以热力色阶呈现（红正绿负），不画曲线。
- 用途：白酒 + 食品饮料相关性 > 0.9 → "伪分散"提示。

### 4.5 组合层最大回撤（P1）

- 按当前持仓份额/权重，回放历史每日：组合日净值 =
  `Σ(单基金 shares × 当日 nav_adj) / 当前总市值`（归一到 1 起步）。
  > 简化口径：假设当前持仓回溯整个区间不变（静态回测），不做动态调仓回测。
- 逐日算峰谷回撤，取 max，返回峰谷日期 + 回撤 %。
- 数据不足（任一持仓 nav_history < 60 天）→ null。

## 五、数据来源与依赖

- `holding` / `fund_transaction`（`compute_position`）/ `fund_nav_history`
  / `fund_list.fund_type`，全部已落库。
- 依赖 PRD-02 的 `nav_adj` 用于相关性、组合回撤；PRD-02 未合时用 `nav` 兜底
  并在 `note` 标注"未复权"。

## 七、前端交互

- 不画组合净值走势曲线（红线）；组合回撤只给数字 + 峰谷日期。

## 八、验收标准

1. 单只持仓时，组合总览 == 单基金盈亏，与详情页数字一致。
2. 多只持仓：`Σ(各 holding 市值) == total_market_value`，`Σ(各 allocation amount) == total`。
3. 全空仓用户：返回空结构不报错（`holdings_count=0`）。
4. 集中度预警：单基金 > 40% 时 `warn=true`，前端标红。
5. 相关性矩阵对角线恒 1.0，对称，样本不足对为 null。
6. `bash scripts/test.sh` 全绿，含 `test_portfolio.py`。

## 九、范围边界（刻意不做）

| 项目 | 状态 |
|---|---|
| 组合净值走势曲线 / 累计收益曲线 | 不画（走势图红线） |
| 动态调仓回测 / 历史持仓回放 | 不做（静态当前持仓回溯，够自用） |
| 目标配置（用户设目标比例 + 再平衡建议） | 不做（属再平衡策略，后续主题） |
| 组合层夏普/波动率 | 不做（与 PRD-01 重复口径，先用组合回撤一个数够用） |
