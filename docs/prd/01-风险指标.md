# PRD-01 基金风险指标(波动率 / 最大回撤 / 夏普 / 索提诺 / 卡玛)

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P0　｜　状态：待立项
>
> 主题：「盈见」从净值查询器升级为基金分析器　｜　分类：收益分析做深

## 一、背景与价值

平台已在 `fund_nav_history` 落库每日单位净值 + 当日涨跌幅（`equity_return`，M8-B），
但 `backend/api/returns.py` 当前仅计算最朴素的区间涨跌幅
`（期末净值 − 期初净值）/ 期初净值`。

**问题**：收益脱离风险毫无意义。一只基金赚 15%，是用了 5% 波动率换来、还是 30%
波动率换来的，对持有体验和决策是两回事。当前平台只有"秤"没有"尺"——只称收益，
不量风险。

**价值**：基于已落库的每日净值序列，补齐 5 项基金风险常识指标。**零新增数据源、
零合规风险、不碰任何红线**（全是点状统计，非走势连续曲线），却能让平台从
"现状查询"长出真正的"分析"能力，契合产品魂「现状 vs 预期」——因为专业"预期"
本就包含"我能承受多大风险"。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 自用主理人 | 打开任一持仓基金详情页，一眼看到"这只最惨跌过多少、波动多大、性价比如何" |
| 选基对比 | 在市场页候选基金之间横向比较风险调整后收益，而非只看绝对涨幅 |
| 止盈止损复盘 | 触发止盈/止损后，回看该基金历史回撤幅度，校准下次目标线是否合理 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 年化波动率 | 日收益率标准差 × √252 | P0 |
| 最大回撤（MDD） | 净值峰值到谷底的最大跌幅 % | P0 |
| 夏普比率 | （年化收益 − 无风险利率）/ 年化波动率 | P0 |
| 索提诺比率 | 仅用下行波动替代总波动 | P1 |
| 卡玛比率 | 年化收益 / 最大回撤 | P1 |

> 计算窗口统一为"近 1 年"，与现有 `returns.py` 的阶段切片对齐；数据不足 1 年的
> 基金返回 null（不强行外推）。

## 四、详细需求（计算口径）

### 4.1 年化波动率 `volatility`

- 输入：近 1 年（≤365 天）的每日 `equity_return` 序列，跳过 null 值。
- 日收益率 `r_t = equity_return / 100`（库中存的是百分数）。
- 样本标准差 `σ_daily = sqrt( Σ(r_t − r̄)² / (n−1) )`，n ≥ 30 才计算，否则 null。
- 年化：`volatility = σ_daily × √252`，保留 2 位小数（百分数形式，如 23.45）。

### 4.2 最大回撤 `max_drawdown`

- 输入：近 1 年每日 `nav`（单位净值），跳过 null。
- 逐日记录截至当日的累计峰值 `peak_t = max(nav_1..nav_t)`。
- 当日回撤 `dd_t = (nav_t − peak_t) / peak_t`。
- `max_drawdown = min(dd_t) × 100`，保留 2 位小数（负数，如 −18.34）。
- 同时返回 `max_drawdown` 对应的 `peak_date` 与 `trough_date`，便于前端标注。

### 4.3 夏普比率 `sharpe`

- 年化收益 `R_annual`：复用 `returns.py` 的 `m1` 同款思路但拉满 1 年窗口，
  `R_annual = (latest_nav / nav_1y_ago − 1) × 100`。
- 无风险利率 `R_f`：默认 2.0%（年化，对应一年期国债大致水平），可通过环境变量
  `FUNDSIGHT_RISK_FREE_RATE` 覆盖；私享自用，写死默认值即可，不必落库。
- `sharpe = (R_annual − R_f) / volatility`，保留 3 位小数。
- 任一依赖项为 null（1 年数据不足）→ sharpe = null。

### 4.4 索提诺比率 `sortino`（P1）

- 仅取负收益日：`σ_downside = sqrt( Σ r_t<0 (r_t − 0)² / n_downside )`，
  年化 `σ_downside × √252`。
- `sortino = (R_annual − R_f) / σ_downside_annual`，保留 3 位。
- n_downside < 12 → null。

### 4.5 卡玛比率 `calmar`（P1）

- `calmar = R_annual / abs(max_drawdown)`，保留 3 位。
- max_drawdown = 0 或 null → calmar = null。

## 五、数据来源与依赖

- **唯一数据源**：`fund_nav_history` 表（已落库，抓取层 `nav_history.py` 日更写入）。
- **不触发任何外部抓取**，沿用 M6/M7「业务层只读缓存」红线。
- 依赖 PRD-02「分红复权净值」先行落地——否则分红日的假大跌会污染波动率与回撤。
  若 PRD-02 未合入，本 PRD 需在分红日做最小化兜底（跳过 |equity_return| 异常大的
  单点，标记数据可疑），但属妥协方案，不推荐。

## 六、API 设计

新增端点（复用 `backend/api/returns.py` 模块，避免新开文件）：

```
GET /api/fund/{code}/risk?window=1y
```

响应：
```json
{
  "fund_code": "020608",
  "window": "1y",
  "volatility": 23.45,
  "max_drawdown": -18.34,
  "max_drawdown_peak_date": "2025-09-24",
  "max_drawdown_trough_date": "2025-10-18",
  "sharpe": 0.82,
  "sortino": 1.15,
  "calmar": 0.91,
  "sample_days": 242,
  "note": null
}
```

- `sample_days`：实际参与计算的交易日数，便于前端判断可信度。
- `note`：数据不足或疑似分红污染时的说明文案，前端可灰字提示。
- 未登录不阻断（风险指标属公开统计维度，同 `get_returns` 现有约定）；
  但若评审认为应隔离，则加 `ctx.user_id is None → 401`，与 cost-curve 对齐。

## 七、前端交互

- 基金详情页（`frontend/detail.js`）基本面卡片下方，新增"风险概览"小卡：
  - 四宫格：波动率 / 最大回撤 / 夏普 / 卡玛（索提诺 P1 再补）。
  - 最大回撤下方灰字标"峰值 09-24 → 谷底 10-18"。
  - 任一指标为 null 显示 "—"，hover 提示"数据不足"。
- **不画走势曲线、不画回撤区间填充图**（守"不画走势图"红线）。回撤的峰谷日期用
  纯文本标注，给的是"点"不是"线"。

## 八、验收标准

1. 用近 1 年有完整净值的种子基金调用接口，5 项指标均有数值，`sample_days` 与实际交易日数一致。
2. 对成立不足 1 年的基金，全部返回 null，`note` 提示"数据不足 1 年"。
3. 人工对照天天基金同类页面，波动率/最大回撤误差 ≤ 0.5 个百分点（口径差异容忍）。
4. 单元测试覆盖：全 null 序列、分红日假大跌、单一峰值、单调下跌（回撤=区间跌幅）等边界。
5. `bash scripts/test.sh` 全绿。

## 九、范围边界（刻意不做，红线）

| 项目 | 状态 |
|---|---|
| 连续走势曲线 / K 线 / 回撤区间填充图 | 仍不做（守"不画走势图"红线） |
| VaR / CVaR / Beta 等需基准的指标 | 不在本 PRD，归 PRD-06 基准对比 |
