# PRD-07 预期深化（年化目标 / 回本所需涨幅 / 移动止盈）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P2　｜　状态：待立项
>
> 主题：延伸产品魂「现状 vs 预期」——给预期加时间轴与回撤意识　｜　分类：组合看板做深

## 一、背景与价值

平台的魂是「现状 vs 预期」。当前"预期"是 `holding.target_rate` 静态目标线，
存在两个体验缺口：
1. **不可比**：持有 3 个月赚 5% 和持有 2 年赚 5%，目标线都是百分比，无法跨周期比较。
2. **无回撤意识**：止损时用户最需要的清醒剂——"亏 20% 要涨 25% 才回本"——平台给不出；
   止盈线是静态的，不会随净值上行而抬升（移动止盈）。

**价值**：纯计算增强，零新数据源，把"预期"从静态百分比升级为年化可比 + 时间推算
+ 回撤自适应。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 持有较久用户 | 目标收益率年化化，跨持仓跨周期横向可比 |
| 亏损用户 | 看到"回本所需涨幅"，破除"亏 20% 再涨 20% 就回本"的直觉错觉 |
| 止盈管理 | 用移动止盈：从高点回撤 X% 自动触发，锁住浮盈而非死等目标线 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 目标年化化 | target_rate → 年化目标，按持有期换算 | P0 |
| 回本所需涨幅 | 当前亏损率 → 回到成本需要的涨幅 % | P0 |
| 达成时间推算 | 按当前增长速度，距目标还差多久 | P1 |
| 移动止盈（回撤止盈） | 从持仓期最高净值回撤 X% 触发，替代静态止盈线 | P1 |

## 四、详细需求

### 4.1 目标年化化

- 现有 `target_rate` 是绝对百分比。新增只读派生字段 `target_annual`：
  `target_annual = (1 + target_rate/100)^(365/holding_days) − 1`，holding_days 从
  `holding.created_at` 至今。
- 前端详情页同时展示绝对目标与年化目标，便于跨持仓对比。
- 不改 `target_rate` 落库口径（只读派生，避免迁移）。

### 4.2 回本所需涨幅

- 当前亏损率 `loss_pct = (current_value − cost) / cost × 100`（负值）。
- 回本所需涨幅 `recovery_pct = (cost / current_value − 1) × 100`，保留 2 位。
- 反直觉常识提示：亏损越深，回本所需涨幅非线性放大（亏 50% 要涨 100%）。
- 仅当 current_value < cost（浮亏）时计算；浮盈时返回 null。

### 4.3 达成时间推算（P1）

- 取该基金近 3 个月 `equity_return_adj` 的年化收益率 `r_proj`。
- 距目标剩余涨幅 `gap = target_rate − current_return_pct`。
- 估算达成所需天数 `days_to_target ≈ ln(1+gap/100+1) / ln(1+r_proj) × 365`
  （仅当 r_proj > 0）。
- r_proj ≤ 0 或数据不足 → null，`note` 标"当前趋势下短期无法达成"。
- **这是推算不是承诺**，前端文案须明确"按近 3 月增速推算，仅供参考"。

### 4.4 移动止盈（P1）

- `holding` 加列 `trailing_stop_pct REAL`（从最高净值回撤触发比例，可空）。
- 维护持仓期最高净值 `peak_nav`（`holding` 加列 `peak_nav REAL`，scheduler 日更
  对比最新 nav 更新）。
- 触发：`当前净值 ≤ peak_nav × (1 − trailing_stop_pct/100)` 且当前浮盈 →
  推 `notification`（kind 新增 `trailing_stop_hit`），并 UI 高亮。
- 与现有静态 `stop_profit` 并存，用户二选一或都用。

## 五、数据来源与依赖

- 全部基于已有 `holding` + `fund_nav_history` + `compute_position`，纯计算。
- 移动止盈依赖 scheduler 日更 `peak_nav`，复用 M9 巡检机制。
- 达成时间推算依赖 PRD-02 复权涨跌幅。

## 六、API 设计

扩展现有持仓接口响应（不新增端点）：
```json
{
  "target_rate": 30.0,
  "target_annual": 12.5,
  "recovery_pct": 25.0,
  "days_to_target_est": 180,
  "trailing_stop_pct": 8.0,
  "peak_nav": 1.452,
  "note": "按近3月增速推算，仅供参考"
}
```

## 七、前端交互

- 持仓卡片目标行：`目标 30% (年化 12.5%)`。
- 浮亏时显示"回本需涨 25%"红字角标。
- 详情页"预期"区：达成时间推算灰字"约还需 6 个月（按当前增速）"。
- 移动止盈：编辑弹窗加"移动止盈回撤 %"输入，触发时高亮 + 站内通知。

## 八、验收标准

1. 年化目标：持有 365 天、target_rate=30% → target_annual=30%；持有 730 天、
   target_rate=30% → target_annual≈14.0%。
2. 回本涨幅：亏 20% → recovery_pct=25.0；亏 50% → 100.0。
3. 浮盈时 recovery_pct=null。
4. 移动止盈：peak_nav=1.452、trailing=8%，净值跌到 1.336 时触发。
5. scheduler 正确更新 peak_nav（只增不减）。
6. `bash scripts/test.sh` 全绿。

## 九、范围边界（刻意不做）

| 项目 | 状态 |
|---|---|
| 达成时间作为"承诺/倒计时"强提示 | 不做（推算有不确定性，文案须弱化） |
| 自动止盈下单 / 触发交易 | 绝不做（合规红线，不碰交易） |
| 目标价 AI 预测 | 不做（越界到预测，自用不必） |

## 十、优先级与里程碑

- 优先级：P0（年化化 + 回本涨幅），P1（达成推算 + 移动止盈）。
- 工作量：后端 1 人日（含 holding 加列 + scheduler 扩展），前端 0.5 人日。
- 依赖：PRD-02 复权涨跌幅（达成推算）。
