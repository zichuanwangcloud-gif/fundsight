# PRD-06 基准对比（Alpha / Beta / 跟踪误差 / 信息比率 / 同类百分位）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P2　｜　状态：待立项（需先评估数据源）
>
> 主题：给"预期"装上基准锚——主动基金能力评价　｜　分类：收益分析做深

## 一、背景与价值

平台魂是「现状 vs 预期」，但"预期"目前只是用户拍的目标线。**金融上基金的预期
必须对标基准**：一只主动基金涨 15%，不知道是跑赢沪深300 还是跑输同类，这 15%
就是裸数据。

**问题**：缺基准对比，主动基金无法评价能力，指数基金无法评价跟踪质量。

**价值**：引入基准净值序列，补 Alpha/Beta/跟踪误差/信息比率/同类百分位，
让"现状 vs 预期"升级为"现状 vs 基准"。

> 本 PRD 涉及新增外部数据源（指数/同类均值），合规上仍属自用私享低频缓存，
> 但比 PRD-01 的纯内部计算重，**建议先做数据源评估子任务再正式立项**。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 主动基金持有者 | 看 Alpha + 同类百分位，判断经理"是真能赚还是吃 beta" |
| 指数/ETF 持有者 | 看跟踪误差，误差大的指数基金直接 pass |
| 选基者 | 市场页按同类排名（百分位）筛选，而非只按绝对收益排 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 基准净值序列落库 | 沪深300/中证500/中证全债等指数日净值 | P0(前置) |
| 超额收益（Alpha 粗口径） | 基金收益 − 基准收益 | P0 |
| Beta | CAPM 回归斜率 | P0 |
| 跟踪误差 / 信息比率 | 指数基金必看 | P1 |
| 同类百分位 | 该基金在同类中的排名分位 | P1 |

## 四、详细需求

### 4.1 基准表

```sql
CREATE TABLE IF NOT EXISTS benchmark_nav (
    benchmark_code TEXT NOT NULL,   -- hs300 / zz500 / csi_allbond ...
    nav_date       TEXT NOT NULL,
    close          REAL,            -- 指数收盘点位
    PRIMARY KEY (benchmark_code, nav_date)
);
CREATE TABLE IF NOT EXISTS fund_benchmark_map (
    fund_code    TEXT PRIMARY KEY,
    benchmark_code TEXT             -- 该基金的默认对标基准
);
```

- 基金默认基准映射：指数/ETF 用其跟踪指数；主动混合用沪深300；债基用中证全债。
  缺省时回落到沪深300，并在 `note` 标注"默认基准"。

### 4.2 Alpha（粗口径）

- 近 1 年 `R_fund`（基金复权收益）、`R_bench`（基准同期收益）。
- `Alpha = R_fund − R_bench`（百分点），保留 2 位。
- 严格 Jensen's Alpha（CAPM 截距）作为 P2 增强，先用粗口径够自用。

### 4.3 Beta

- 取基金与基准近 1 年日 `equity_return_adj` 对齐序列（重叠 ≥ 60 日）。
- `Beta = Cov(r_fund, r_bench) / Var(r_bench)`，保留 3 位。

### 4.4 跟踪误差 / 信息比率（P1）

- 跟踪误差 `TE = std(r_fund − r_bench) × √252`（年化，百分点）。
- 信息比率 `IR = (R_fund − R_bench) / TE`。
- 主要服务指数/ETF；主动基金 IR 也可算，口径同。

### 4.5 同类百分位（P1）

- 取该基金 `fund_type` 对应大类（复用 `market.py` 归类），从 `fund_profile`
  读该类所有基金的 `syl_1n`（近 1 年收益），算该基金在该类中的百分位
  （0-100，越大越靠前）。
- 依赖 `fund_profile.syl_1n` 已全量同步；同类样本 < 20 时返回 null 标注"样本不足"。

## 五、数据来源与依赖

- **基准指数日净值**：东财指数行情接口或 AKShare `index_zh_a_hist`，收敛到
  `backend/datasource/` 新增 `benchmark_nav.py`，低频日更，守 M6 红线。

- 依赖 PRD-02 复权净值（Beta/TE 用 `equity_return_adj`）。

## 七、前端交互

## 八、验收标准

1. ETF 联接基金 Beta ≈ 0.9~1.0，跟踪误差小（< 2pp）。
2. 主动混合 Alpha 正负合理，与天天基金同类排名方向一致。
3. 同类百分位与天天基金同类排名换算误差 ≤ 5 个百分位。
4. 基准缺失时回落沪深300，`benchmark_is_default=true`。
5. `bash scripts/test.sh` 全绿，含 `test_benchmark.py`。

## 九、范围边界（刻意不做）

| 项目 | 状态 |
|---|---|
| 多因子归因（Brinson/Fama） | 不做（自用私享不必上归因模型） |
| 自定义基准组合 | 不做（写死映射，私用够用） |
| 选股择时能力分解（Treynor-Mazuy） | 不做（过度专业） |
