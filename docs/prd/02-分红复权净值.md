# PRD-02 分红复权净值（修真实盈亏 / 回撤计算正确性）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P0　｜　状态：待立项
>
> 主题：收益/风险计算的正确性根基　｜　分类：数据可靠性收口

## 一、背景与价值

当前 `fund_nav_history` 存的是**单位净值**（`nav`），`equity_return` 为当日涨跌幅。
所有盈亏、阶段收益、（PRD-01 的）波动率与回撤，都基于这一列计算。

**问题**：基金**分红日单位净值会断崖式下跌**——钱分给持有人了，净值不是真跌。
当前口径下：
- 分红日的 `equity_return` 会是"假大跌"（如 -8%），污染波动率与最大回撤；
- 跨分红期算阶段收益 `(期末−期初)/期初`，把分红错算成了亏损；
- `compute_position` 的真实盈亏会把分红当亏损。

**这是正确性 bug，不是新功能**。不修则 PRD-01 的所有风险数、现有 returns 的
阶段收益、交易流水的真实盈亏，在跨分红期时全部可能失真。优先级高于任何新指标。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 所有持仓用户 | 持有的基金分红后，看到的是"真实"回撤与收益，而非被分红吓到的假跌 |
| 风险指标消费者 | PRD-01 的波动率/回撤不再被分红日污染 |

## 四、详细需求

### 4.1 分红数据表

```sql
CREATE TABLE IF NOT EXISTS fund_dividend (
    fund_code   TEXT NOT NULL,
    ex_date     TEXT NOT NULL,   -- 权益登记日 / 除息日（取除息日，净值跳变日）
    per_share   REAL,            -- 每份分红（元）
    type        TEXT,             -- cash 现金分红 | reinvest 红利再投
    record_date TEXT,             -- 权益登记日
    payout_date TEXT,             -- 派发日
    PRIMARY KEY (fund_code, ex_date)
);
```

### 4.2 复权净值口径

采用**前复权（向前调整）**：以最新净值为基准，向历史回溯，每个除息日把此前所有
净值按分红比例上调。

- 单次分红调整因子 `f = 1 + per_share / nav_on_ex_date`。
- 该除息日**之前**的每条 nav 乘以累计因子：`nav_adj = nav × ∏ f`。
- 复权后的 `equity_return_adj`：用 `nav_adj` 重算当日涨跌幅，分红日不再是假大跌。

> 选前复权而非后复权：前复权以最新值为基准，与用户"现在净值多少"心智一致，
> 便于详情页走势散点从左到右递增。

### 4.3 计算时机

- 不实时算。抓取层 `nav_history.refresh_nav_history` 写入 nav 后，若
  `fund_dividend` 有该基金记录，则同步重算 `nav_adj` 与 `equity_return_adj`
  并更新回 `fund_nav_history` 两列（新增列，见 4.4）。
- 业务层只读 `nav_adj`，沿用只读红线。

### 4.4 表结构增量

`fund_nav_history` 加两列（沿用 `_ensure_columns` 幂等迁移）：
- `nav_adj REAL`——前复权净值；
- `equity_return_adj REAL`——复权涨跌幅 %。

历史存量数据：首次启用时跑一次全量回填脚本，按各基金 `fund_dividend` 回算。

## 五、数据来源与依赖

- 分红记录：东财 `pingzhongdata` 含分红送配字段（与 `fund_profile` 同源），
  收敛到 `backend/datasource/` 新增 `fund_dividend.py`。
- 低频：随 profile 同批刷新（已低频），不新增抓取频次。
- 合规：同 fund_profile，自用私享低频缓存，不碰红线。

## 六、API 设计

**本 PRD 不新增对外 API**，是对底层数据口径的修正：

- `fund_detail._read_series` 返回的 series 增加 `nav_adj`、`equity_return_adj`、
  `is_dividend_day`（布尔，前端打标用）。
- `returns._compute_periods` 改读 `nav_adj`。
- `returns._attribution_for_period` 的 `end_nav` 改读 `nav_adj`。
- `compute_position` 真实盈亏若依赖 nav 比较，改读 `nav_adj`。
- PRD-01 全部改读 `equity_return_adj` 与 `nav_adj`。

仅新增一个内部回填端点（管理员/手动触发，不计入正式路由）：
```
POST /api/admin/rebuild-adj-nav?code=   # 单基金回填复权
```
需登录，复用 M9 系统状态页的 admin 约定。

## 七、前端交互

- 详情页走势散点：分红日点用小三角标注，hover 显示"分红 X 元/份"。
- 不新增页面、不画分红累计曲线。
- 风险概览卡片（PRD-01）数据自动变准，前端无感。

## 八、验收标准

1. 取一只有分红记录的基金，分红日 `equity_return` 为负大跌，`equity_return_adj`
   接近 0 或正常波动，二者差异 ≈ 分红率。
2. 跨分红期阶段收益：用 `nav_adj` 算的结果 vs 天天基金复权收益，误差 ≤ 0.5pp。
3. 最大回撤（PRD-01）不再在分红日出现假谷底。
4. 回填脚本对全量持仓基金幂等可重跑，不产生重复列或报错。
5. `bash scripts/test.sh` 全绿，含新增 `test_dividend_adj.py`。
