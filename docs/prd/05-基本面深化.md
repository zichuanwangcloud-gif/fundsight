# PRD-05 基金基本面深化（重仓股 / 行业分布 / 换手率 / 经理维度 / 评级）

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P1　｜　状态：待立项
>
> 主题：详情页专业度提升　｜　分类：抓取层扩面

## 一、背景与价值

`fund_profile` 当前只有 经理 / 规模 / 费率 / 4 个收益率，详情页（`fund_detail.py`）
基本面卡片信息密度低。专业选基金第一眼看的是**重仓股和行业分布**，平台全缺。

**问题**：用户想判断"这只消费基金是不是伪装的白酒基金"，看不到重仓股；
想判断经理"长拿还是炒短"，看不到换手率；想按经理维度串起管理过的基金，
平台有 `manager` 字段却没用起来。

**价值**：扩 `fund_profile` 抓取字段，详情页补专业维度，让选基有依据。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 选基者 | 详情页看重仓股前十 + 行业分布，判断"伪分散 / 主题漂移" |
| 经理研究者 | 按经理查其管理的所有基金 + 任职回报，识破"挂名经理" |
| 费率敏感者 | 看换手率（高的隐性成本高） + 持有期赎回费阶梯 |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 前十大重仓股 + 集中度 | 重仓股名/占比、CR10 | P0 |
| 行业分布 | 按申万一级行业占比 | P0 |
| 换手率 | 近一年换手率 | P1 |
| 经理维度页 | 经理管理的基金列表 + 任职回报 | P1 |
| 评级 | 晨星/银河评级（3/4/5 星） | P2 |
| 机构持有比例 / 份额申赎动向 | 资金流向参考 | P2 |

## 四、详细需求

### 4.1 重仓股与行业

- 抓取层 `fund_profile` 扩展，落新表：
```sql
CREATE TABLE IF NOT EXISTS fund_holdings (
    fund_code   TEXT NOT NULL,
    report_date TEXT NOT NULL,    -- 季报截止日
    stock_code  TEXT,
    stock_name  TEXT,
    weight      REAL,              -- 占净值比
    rank        INTEGER,
    PRIMARY KEY (fund_code, report_date, stock_code)
);
CREATE TABLE IF NOT EXISTS fund_industry (
    fund_code   TEXT NOT NULL,
    report_date TEXT NOT NULL,
    industry    TEXT,
    weight      REAL,
    PRIMARY KEY (fund_code, report_date, industry)
);
```
- 详情页取最新 `report_date` 一组，CR10 = Σ top10 weight。

### 4.2 换手率（P1）

- `fund_profile` 加列 `turnover REAL`（近一年换手率 %），抓取层同批写入。
- 无则 null，前端显示 "—"。

### 4.3 经理维度页（P1）

- 复用 `fund_profile.manager`，新增端点反查：
  ```
  GET /api/manager?name=张三
  → 该经理名下所有基金（fund_code/name/scale/任职回报/任职年限）
  ```
- 经理名可能重名，私享自用按字符串精确匹配即可，不做实体消歧。

### 4.4 评级（P2）

- `fund_profile` 加列 `rating_morningstar INTEGER`（1-5，可空）。
- 东财 pingzhongdata 有评级字段，抓取层低频写入。

## 五、数据来源与依赖

- 全部来自东财 `pingzhongdata`（已与 `fund_profile` 同源），收敛到
  `backend/datasource/fund_profile.py` 扩展，不新增对外接口。
- 低频：随 profile 刷新批，不增加抓取频次，守 M6 红线。
- 合规：自用私享低频缓存，不碰红线。

## 六、API 设计

详情端点扩展响应（`GET /api/fund/{code}`）：
```json
{
  "profile": {"manager":"...", "scale":..., "turnover": 1.85, "rating_morningstar": 4, ...},
  "series": [...],
  "top_holdings": [
    {"rank":1,"stock_code":"600519","stock_name":"贵州茅台","weight":9.8},
    ...
  ],
  "industry": [{"industry":"食品饮料","weight":68.2}, ...],
  "holdings_report_date": "2026-03-31"
}
```
新增经理端点：`GET /api/manager?name=`。

## 七、前端交互

- 详情页基本面卡片扩展为多 tab：概览 / 重仓 / 行业 / 经理。
- 重仓 tab：前十表格 + 集中度条；行业 tab：占比条形图（非曲线，条形可）。
- 经理 tab：管理基金列表，点基金跳详情。
- 评级用 ★ 展示，无则空。

## 八、验收标准

1. 取一只有季报的基金，重仓股前十与天天基金一致（按最新季报）。
2. 行业占比 Σ ≈ 100%（股票型/混合型；债基可不满足，标注"债基不适用"）。
3. 换手率字段缺失返回 null，前端 "—"。
4. 经理端点按名精确匹配，重名场景标注"可能重名，请核对"。
5. `bash scripts/test.sh` 全绿。

## 九、范围边界（刻意不做）

| 项目 | 状态 |
