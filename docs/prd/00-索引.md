```
02 分红复权 ──┬──► 01 风险指标（波动率/回撤读 equity_return_adj）
              ├──► 03 组合层汇总（相关性/组合回撤读 nav_adj）
              ├──► 04 定投模拟（跨分红期收益依赖复权）

              └──► 07 预期深化（达成推算读复权涨跌幅）

03 组合层汇总 ── 依赖 02（P1 部分）；P0 部分可独立先上
06 基准对比  ── 前置：数据源评估子任务
08 指数估值  ── 前置：数据源 + 合规复评（红线最敏感）
```

- **02 是正确性根基，应先于或同期于 01 合入**。
- 03 的 P0 部分（总览/配置/集中度）不依赖 02，可独立先上。
- 04/05/06/07 互不冲突，文件集不相交，可并行 worktree 推进。

## 红线复评

本批次所有 PRD 均在 M9 立项红线之内：

- 不画走势连续曲线 / K 线 —— 全部用点状统计、进度条、条形、热力色阶呈现。
- 不做 Web Push / 手机推送 —— 定投提醒复用 M9-d 站内角标口径。
- 不碰交易 / 不接券商 API —— 止盈止损仅高亮 + 通知，不下单。
- 自用私享低频缓存 —— 抓取层低频，业务层只读 SQLite。

> ⚠️ PRD-08（指数估值）合规边界比净值紧，公开/收费前须重走合规评估，
> 届时 Out-of-Scope 红线须整体重审（同立项文档红线复评口径）。

## 建议推进顺序

1. **第一波（正确性 + 分析底座）**：02 分红复权 → 01 风险指标 → 03 组合层汇总 P0
2. **第二波（核心玩法 + 详情深化）**：04 定投 + 05 基本面 + 07 预期深化
3. **第三波（需评估）**：06 基准对比（评估通过后）→ 08 指数估值（合规复评后）
