# PRD-04 定投模拟与定投计划

> 版本：v1.0　｜　日期：2026-07-13　｜　优先级：P1　｜　状态：待立项
>
> 主题：基金核心玩法 DCA（Dollar-Cost Averaging）落地　｜　分类：收益分析做深

## 一、背景与价值

基金区别于股票的核心玩法是**定投**。平台已有买卖流水与加权成本推导，但完全没有
定投场景：不能回测定投收益、不能管理定投计划、不能对比定投 vs 一次性。

**问题**：用户想"每月定投 X 元、坚持 1 年、收益如何"，平台答不出。

**价值**：基于 `fund_nav_history` 回测 DCA 收益率与微笑曲线区间，复用 M9-d 站内
通知做定投到点提醒。这是基金用户最高频的真实需求。

## 二、目标用户与场景

| 用户 | 场景 |
|---|---|
| 定投决策者 | 输入"每月 1000、起止日期"，看该基金历史区间定投收益率、微笑区间 |
| 对比决策 | 同区间定投 vs 一次性投入，看哪种赚得多（定投通常在下跌市占优） |
| 计划持有者 | 设定每月定投日 + 金额，到点站内提醒"该定投了" |

## 三、功能范围

| 能力 | 说明 | 优先级 |
|---|---|---|
| 定投模拟器（回测） | 给定区间+金额+频率，回算 DCA 收益率 | P0 |
| 定投 vs 一次性对比 | 同区间两种策略结果并列 | P0 |
| 定投计划 CRUD | 设定/编辑/删除定投计划 | P1 |
| 定投到点提醒 | 复用 M9-d 通知，定投日推站内角标 | P1 |

## 四、详细需求

### 4.1 定投模拟器口径

- 输入：`fund_code`、`start_date`、`end_date`、`per_amount`（每期金额）、
  `freq`（weekly/biweekly/monthly）、`invest_day`（每月几号/每周几）。
- 每期按 `invest_day` 取当日 `nav_adj`（复权净值，依赖 PRD-02；未合时用 nav 兜底
  标注），买入份额 = `per_amount / nav_adj`。
- 若当日无净值（节假日），顺延到下一个有净值日。
- 汇总：`total_invested`、`total_shares`、`final_value = total_shares × end_nav_adj`、
  `dca_return_pct = (final_value − total_invested) / total_invested × 100`、
  `annualized_return`（按投入时长年化）。
- 微笑曲线区间：返回区间内 `nav_adj` 的 min/max 及对应日期（点状统计，**不画
  连续走势曲线**），供前端用文字 + 两个点呈现"低点在 X 月、高点在 Y 月"。

### 4.2 定投 vs 一次性对比

- 一次性投入：`total_invested`（= Σ 各期金额）在 start_date 一次性买入，
  `final_value_lump = (total_invested / nav_start) × end_nav_adj`。
- 输出 `lump_return_pct` 与 `dca_return_pct`，以及 `diff`（DCA − 一次性）。
- 不预设立投更好/定投更好，只给数。

### 4.3 定投计划数据表

```sql
CREATE TABLE IF NOT EXISTS dca_plan (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL,
    fund_code   TEXT NOT NULL,
    per_amount  REAL NOT NULL,
    freq        TEXT NOT NULL,        -- weekly | biweekly | monthly
    invest_day  INTEGER NOT NULL,     -- 每月几号(1-28) 或每周几(0-6)
    next_date   TEXT NOT NULL,        -- 下次触发日
    active      INTEGER DEFAULT 1,
    created_at  TEXT
);
CREATE INDEX IF NOT EXISTS idx_dca_plan_user ON dca_plan(user_id, active);
```

### 4.4 定投到点提醒

- `scheduler.py` 日更巡检 `dca_plan` 中 `next_date <= today` 的活跃计划，
  推 `notification`（kind 新增 `dca_due`），消息"该给 XX 基金定投 ¥X 了"。
- 推送后 `next_date` 按频率滚到下一期。
- 仅站内角标（M9-d 口径），不做 Web Push / 手机推送（红线）。

## 五、数据来源与依赖

- 模拟：`fund_nav_history.nav_adj`（依赖 PRD-02），纯只读。
- 计划：新增 `dca_plan` 表，用户私有数据。
- 提醒：复用 `notification` 表 + `scheduler` 日更。

## 六、API 设计

```
GET  /api/fund/{code}/dca-simulate?start=&end=&amount=&freq=&invest_day=
POST /api/dca/plans            # 增
GET  /api/dca/plans            # 列当前用户
PUT  /api/dca/plans/{id}       # 改
DELETE /api/dca/plans/{id}     # 删
```

## 七、前端交互

- 详情页新增"定投回测"折叠区：起止日期 + 每期金额 + 频率 → 运行，展示
  DCA 收益率、vs 一次性、低点/高点日期（点状文字，不画曲线）。

  到点未执行有角标。

## 八、验收标准

1. 单调上涨区间：定投收益 < 一次性收益（`diff` 为负）。
2. V 型区间（先跌后涨）：定投收益 > 一次性（`diff` 为正，微笑曲线）。
3. 节假日 invest_day 自动顺延到下一有净值日，份额正确。
4. 计划 CRUD 按 user_id 隔离，越权改删不生效。
5. scheduler 到点推 `dca_due` 通知，next_date 正确滚动。
6. `bash scripts/test.sh` 全绿，含 `test_dca.py`。

## 九、范围边界（刻意不做）

| 项目 | 状态 |
|---|---|
| 定投收益走势曲线 / 累计收益曲线 | 不画（走势图红线，只给低点高点点状） |
| 智能定投（按估值/均线动态调额） | 不做（属策略，自用够用即可） |
| 定投自动下单 / 接券商 API | 绝不做（合规红线，自用私享不碰交易） |
| 红利再投收益对比 | 不做（口径复杂，后续主题） |
