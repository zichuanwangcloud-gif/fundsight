# 主线 A：收益分析深化（M10A-returns-deep）

> 代码名：`M10A-returns-deep`　｜　分支：`feat/m10a-returns-deep`
> 前置：M9-f 阶段收益统计已合并（PR #25），`backend/api/returns.py` 已存在。

## 1. 概述

在 M9-f 点状阶段收益基础上，做**不破"走势历史图"红线**的收益深化：分批买入成本曲线
+ 阶段收益归因。核心价值：让用户看清"每次买入对总收益的贡献"，辅助加仓/止盈决策。
目标用户：自用 + 朋友（私享级）。

## 2. 需求与用户故事

| # | 需求 | 验收标准 |
|---|---|---|
| A1 | 分批买入成本曲线 | 详情页展示"每次买入后的加权成本"点列（基于 `fund_transaction`），数据不足时该段隐藏 |
| A2 | 阶段收益归因 | 把"近1月/3月/今年以来"收益拆解到各买入批次贡献，返回每批次贡献额与贡献占比 |
| A3 | 数据不足优雅降级 | 无 `fund_transaction` 记录或净值起点缺失时，对应值为 `null`，前端不报错 |
| A4 | 只读不落表 | 全部基于已有 `fund_transaction` + `fund_nav_history` 只读计算，**不新增 daily 快照表** |

## 3. 功能验收清单

- [ ] `GET /api/fund/{code}/cost-curve` 返回买入批次 + 加权成本点列
- [ ] `GET /api/fund/{code}/returns-attribution` 返回各阶段各批次贡献
- [ ] 无交易记录、无净值起点的基金返回 `null` 占位（A3）
- [ ] 详情页渲染成本曲线卡 + 归因卡，复用 `d-grid` 样式，涨跌着色
- [ ] 新增测试覆盖：无数据 / 全数据 / 不足 / missing code

## 4. 技术约束

- **数据模型**：不新增表。读 `fund_transaction`（已存在，M8-c）+ `fund_nav_history`（已存在）。
- **API**：在 `backend/api/returns.py` 的 `ROUTES` 列表追加两项，无需改 `__init__.py`（已注册）。
  - `GET /api/fund/{code}/cost-curve` → `{fund_code, points:[{date, cost_basis, weighted_price}]}`
  - `GET /api/fund/{code}/returns-attribution` → `{fund_code, periods:{m1:{batches:[...]}, ...}}`
  - 需登录（`ctx.user_id is None` → 401），但数据非按用户隔离（基金维度），仅校验登录态。
- **计算**：加权成本 = 累计买入金额 / 累计份额；批次贡献 = 批次份额 × (期末净值 − 批次成本)。
- **前端**：`detail.js` 异步加载两张卡，SVG 点列（沿用 M7 迷你走势图风格，**不画 K 线**）。
- **零依赖**：纯标准库 `sqlite3` + 前端原生 JS。

## 5. 排除项（Out of Scope）

- 收益走势历史图（K 线 / 连续曲线）——仍 Out of Scope，A 只做点状/分批，**不画连续曲线**。
- daily 收益快照表落库。
- 跨基金组合归因（本主线只做单基金）。
