# 主线 B：多用户正式化收口（M10B-auth-hardening）

> 代码名：`M10B-auth-hardening`　｜　分支：`feat/m10b-auth-hardening`
> 前置：M9-e 鉴权加固已合并（PR #24），Secure cookie + 过期 session 清理已落地。

## 1. 概述

M9-e 之后补三件套：接口限流、token 主动吊销、登录审计。核心价值：让"分享给更多朋友"
前的安全基线到位，堵住"私享级登录"暴露面。目标用户：自用 + 朋友（仍私享，不对外运营）。

## 2. 需求与用户故事

| # | 需求 | 验收标准 |
|---|---|---|
| B1 | 接口限流 | 按 `user_id + 端点` 限频（自用级宽松阈值，如 60 次/分钟），超限返回 429 |
| B2 | token 主动吊销 | 改密 / 主动登出时让该用户**所有存量 session** 失效（当前只删当前 session） |
| B3 | 登录审计 | 记录登录成功/失败到 `login_audit` 表（user_id/ip/ua/ok/created_at） |
| B4 | 越权审计只读接口 | `GET /api/admin/login-audit` 仅供登录用户查自己的登录记录（按 user_id 隔离） |

## 3. 功能验收清单

- [ ] 高频请求触发 429（B1）
- [ ] 改密后旧 token 失效，旧 Cookie 请求返回 401（B2）
- [ ] 登录成功/失败均落 `login_audit`（B3）
- [ ] 用户只能查自己的登录记录，越权返回 404（B4）
- [ ] 新增测试覆盖：限流触发 / 吊销 / 审计落库 / 越权

## 4. 技术约束

- **数据模型**（追加到 `db.py` schema 末尾，`notification` 表之后，`-- M10B` 注释块）：
  ```sql
  CREATE TABLE IF NOT EXISTS login_audit (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      user_id INTEGER NOT NULL,
      ip TEXT,
      ua TEXT,
      ok INTEGER NOT NULL,        -- 1 成功 / 0 失败
      created_at TEXT NOT NULL
  );
  CREATE TABLE IF NOT EXISTS rate_limit_state (
      user_id INTEGER NOT NULL,
      endpoint TEXT NOT NULL,
      window_start TEXT NOT NULL,
      count INTEGER NOT NULL,
      PRIMARY KEY (user_id, endpoint, window_start)
  );
  ```
- **限流**：`backend/auth.py` 加 `check_rate_limit(user_id, endpoint)`，内存计数 + SQLite
  落盘兜底（防重启）。阈值宽松，自用级。`app.py` 在 `_current_user()` 之后、分发之前调用。
- **token 吊销**：`backend/auth.py` 加 `revoke_user_sessions(user_id)`，改密/登出时调用，
  删该 user 所有 session 行。
- **路由**：`login_audit` 查询走 `ALL_ROUTES` 扩展点——新增 `backend/api/login_audit.py`
  导出 `ROUTES`，在 `__init__.py` 锚点追加一行（**唯一需要改 `__init__.py` 的主线**，
  追加一行不与 A/C 冲突，A/C 不动 `__init__.py`）。
- **`_router.py`**：**不改 `dispatch` 函数体**，如需中间件以独立函数添加。
- **前端**：`auth.js` 改密成功后提示"已退出其他设备"。
- **零依赖**。

## 5. 排除项

- OAuth / 第三方登录、邮箱验证、找回密码——仍 Out of Scope（私享级自助注册够了）。
- 对外正式注册流程、付费墙——合规红线，不碰。
- 接口签名/加密传输——HTTPS 部署层解决，不在应用层做。
