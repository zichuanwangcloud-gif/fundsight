# 盈见 FundSight —— 架构设计

> 版本：v1.0　｜　日期：2026-07-02

## 设计原则

1. **缓存优先**：用户永远读本地缓存，不直接触发外部接口 —— 既防封 IP，又扛并发。
2. **估值与净值分离**：盘中用估值（参考），收盘用官方净值（准）重算真实盈亏。
3. **数据源可替换**：抓取逻辑收敛到 `datasource/` 一层，未来换源不影响上层。
4. **先单人后多人**：M1 不做登录，数据模型预留 `user_id` 字段，M3 再启用。

## 搜索设计（2026-07-02 新增）

产品要能搜全市场基金（约 1.8 万只）。**绝不实时打外部接口搜索**——已实测东财
全量列表接口 `fund.eastmoney.com/js/fundcode_search.js` 有反爬（418），搜索建议
接口也不稳。正确做法：

```
全量基金列表（代码 + 名称 + 拼音 + 类型）
   │  后台低频同步（每天/每周一次）
   ▼
本地 fund_list 表（SQLite）  ←── 用户搜索只查这张本地表
                                  （毫秒级、永不被封、离线可用）
```

- **同步来源**：`fund.eastmoney.com/js/fundcode_search.js`（全量一次拉齐），
  或 AKShare `fund_name_em()`。二者底层同为东财。
- **降级**：接口拉取失败不影响已有本地库，用户照常搜索旧数据。
- **搜索方式**：本地 `LIKE` 匹配代码 / 名称 / 拼音首字母，小数据量 SQLite 足够快。

> ⚠️ 环境备注：调研沙箱对 `fund.eastmoney.com` 反爬拦截（418）、`fundsuggest` 超时，
> 但 `fundgz.1234567.com.cn`（估值）可达。全量列表同步需在**能正常联网的部署环境**
> 执行；开发期先用**种子数据**（几十只常见基金）跑通完整交互。

## 数据模型（初版，SQLite）

```sql
-- 全量基金列表：搜索用，后台低频同步（约 1.8 万条）
CREATE TABLE fund_list (
    fund_code   TEXT PRIMARY KEY,   -- 基金代码
    name        TEXT,               -- 名称
    pinyin      TEXT,               -- 拼音首字母（搜索用）
    fund_type   TEXT,               -- 类型（股票/混合/债券/QDII/指数…）
    synced_at   TEXT                -- 同步时间
);

-- 行情缓存：抓取层定时写，业务层只读
CREATE TABLE fund_quote (
    fund_code   TEXT PRIMARY KEY,   -- 基金代码
    name        TEXT,               -- 名称
    dwjz        REAL,               -- 昨日单位净值
    gsz         REAL,               -- 盘中估算净值
    gszzl       REAL,               -- 盘中估算涨跌幅 %
    gztime      TEXT,               -- 估值时间
    nav_date    TEXT,               -- 官方净值日期（收盘后回填）
    updated_at  TEXT                -- 本地更新时间
);

-- 自选 + 持仓 + 预期（M1 单人，user_id 预留给 M3）
CREATE TABLE holding (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id       INTEGER DEFAULT 0,  -- M3 多用户启用
    fund_code     TEXT NOT NULL,
    hold_amount   REAL,               -- 当前持仓金额
    cost_amount   REAL,               -- 买入成本（可选）
    shares        REAL,               -- 份额（可由金额/净值反推）
    target_rate   REAL,               -- 预期收益率 %（预期设定）
    target_price  REAL,               -- 目标净值（预期设定）
    stop_profit   REAL,               -- 止盈线
    stop_loss     REAL,               -- 止损线
    created_at    TEXT
);
```

## 关键计算逻辑

- **今日浮动盈亏** = 份额 × (gsz − dwjz)，份额 = 持仓金额 / dwjz
- **真实盈亏**（收盘后） = 份额 × 当日官方净值 − 成本
- **距预期差距** = 目标价 − 当前估值（或 当前收益率 − 目标收益率）
- **止盈止损触发** = 当前收益率 ≥ 止盈线 或 ≤ 止损线 → UI 高亮

> 计算原型已在 `scripts/fund_spike.py` 跑通验证。

## 合规边界（写进架构，长期约束）

- 抓取层**低频 + 缓存**，绝不高频轮询。
- 产品**止步于自用私享**；公开注册/收费前必须重新评估数据授权与合规。
- 数据源声明"仅供学术研究"，不做数据转售、不对外提供数据接口。

## 里程碑对应的实现顺序

1. **M1**：`datasource/fundgz.py` + `akshare_nav.py` + SQLite 建表 + 一个读缓存的 API + 单页前端
2. **M2**：止盈止损计算 + UI 高亮
3. **M3**：登录 + `user_id` 数据隔离
